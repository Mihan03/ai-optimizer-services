# ai-optimizer-services



## MVP

1) Пользователь выбирает кнопку "оптимизировать резюме"
2) Выбирает профессию: пока что будет доступно только java-разработчик
3) Прикладывает свой txt-файл из hh
4) На выходе получает ответ с рекомендациями по исправлению и готовое резюме

Функциональные требования:
1. Ключевые слова парсятся раз в день по 100 вакансиям hh
2. Используется модель gemini 2.5 flash
3. Ответ нейросети будет возвращать json-ответ, по ключам которого будет составлен итоговый ответ. Также в json будет ключ для рекомендаций по резюме
4. В будущем список вакансий будет расширяться
5. В будущем добавим автоотклик на вакансии с составлением автоматического сопроводительного для каждой вакансии

Таблица 1: professions (Профессии)
Хранит уникальные профессии, которые вы анализируете.
Поле	Тип	Описание
id	BIGINT PRIMARY KEY	Уникальный идентификатор
name	VARCHAR(255) UNIQUE	Название профессии ("Java-разработчик")
created_at	TIMESTAMP	Дата создания записи
updated_at	TIMESTAMP	Дата последнего обновления топа навыков

Таблица 2: skills (Навыки)
Это будет ваш "словарь" всех уникальных навыков, которые вы когда-либо встречали.
Поле	Тип	Описание
id	BIGINT PRIMARY KEY	Уникальный идентификатор
name	VARCHAR(255) UNIQUE	Название навыка ("Spring Boot", "PostgreSQL")

Таблица 3: profession_skills (Связующая таблица)
Это "клей", который соединяет профессии и навыки. Каждая строка здесь означает: "Эта профессия требует этот навык".
Поле	Тип	Описание
profession_id	BIGINT	Внешний ключ, ссылается на professions.id

Технологии:
Java, Gradle, Liquibase, PostgreSQL, Spring Boot, Spring Security, Spring Cloud, Spring Data JPA, Kafka, Docker, Kubernetes (опционально), MapStruct, Testcontainers, Redis.
